Introduction

TypeScript is a strongly typed, object-oriented, compiled language. Developed and maintained by Microsoft, it is a strict syntactical superset of JavaScript and adds optional static typing to the language. TypeScript is designed for the development of large applications and transcompiles to JavaScript. This means any valid JavaScript code is also valid TypeScript code, making TypeScript both versatile and powerful for developers who wish to write cleaner, more maintainable code.

TypeScript contains a standard library of objects, types, and interfaces, along with a rich set of language elements such as enums, tuples, and access modifiers. It is often used in conjunction with modern JavaScript frameworks, providing enhanced tooling and static analysis. Additionally, TypeScript can be extended for various purposes:

  • Client-side TypeScript: Just like JavaScript, TypeScript can be used to build complex front-end applications. However, TypeScript’s static type checking helps catch errors early during development, leading to more robust and maintainable code.
  • Server-side TypeScript: TypeScript is also commonly used with Node.js to write server-side code. TypeScript’s modules and interfaces make it an excellent choice for large-scale server applications that require clear structure and scalability.

What you should already know

This guide assumes you have the following basic background:

  • A general understanding of the Internet and the World Wide Web (WWW).
  • Good working knowledge of JavaScript.
  • Some experience with object-oriented programming. If you are new to TypeScript or programming in general, it’s recommended to start with basic JavaScript tutorials before moving on to TypeScript.

TypeScript and JavaScript

TypeScript and JavaScript are closely related but differ in a few key areas. While TypeScript builds on JavaScript by adding optional static types, it retains JavaScript’s dynamic and flexible nature. The addition of types allows developers to specify the data types of variables and function returns, helping prevent type-related errors that can occur in JavaScript.

TypeScript also introduces interfaces and type aliases, which provide a way to define and enforce the structure of objects. Unlike JavaScript’s prototype-based inheritance, TypeScript supports a class-based object model, similar to traditional object-oriented programming languages like Java or C#. TypeScript classes include features like inheritance, access modifiers (public, private, protected), and abstract classes.

Moreover, TypeScript is compiled to JavaScript. The TypeScript compiler checks for type errors and converts TypeScript code into clean, readable JavaScript that can run on any browser or JavaScript engine.

Hello World

To get started with writing TypeScript, you can create a simple "Hello World" program:

function greet(name: string): void { console.log(`Hello, ${name}`); } greet("World");

This function takes a string as an argument and logs a greeting message to the console. The : string annotation specifies that the name parameter must be a string, and : void indicates that the function does not return a value.

Variables

Variables in TypeScript can be declared using let, const, or var. However, using let and const is recommended due to their block scope and immutability, respectively.

A TypeScript identifier must start with a letter, underscore (_), or dollar sign ($); subsequent characters can also be digits (0-9). TypeScript is case-sensitive, so Name and name would be considered different identifiers.

Declaring Variables

You can declare variables in TypeScript similarly to JavaScript, but with optional type annotations:

let isDone: boolean = false; let count: number = 42; let name: string = "John";

TypeScript’s type annotations make it clear what type of data each variable should hold, helping to prevent errors during development.

Variable Scope

TypeScript follows the same scoping rules as JavaScript, with additional support for block-scoped variables using let and const. A variable declared outside of any function is considered global, while variables declared inside a function or block are local to that function or block.

if (true) { let x = 5; console.log(x); // 5 } console.log(x); // Error: x is not defined

In this example, x is scoped to the block, so trying to access it outside the block results in an error.

Constants

TypeScript allows you to declare read-only constants using the const keyword. Once a constant is assigned a value, it cannot be changed.

const PI: number = 3.14;

Data Types

TypeScript supports several basic data types, many of which are similar to JavaScript:

  • boolean - Represents true or false values.
  • number - Represents both integer and floating-point numbers.
  • string - Represents text data.
  • null and undefined - Represent absence of value.
  • any - Represents any data type, useful for dynamic content.
  • void - Represents the absence of any type, usually used for functions that do not return a value.

Additionally, TypeScript introduces new types like:

  • tuple - Represents an array with a fixed number of elements with specific types.
  • enum - A way to define a set of named constants.
  • unknown - Similar to any, but safer and requires explicit type checking before being used.
  • never - Represents values that never occur, typically used for functions that never return.

if...else statement

TypeScript’s if...else statement is identical to JavaScript’s:

if (condition) // statement if condition is true else { // statement if condition is false }

while statement

The while loop in TypeScript functions just like in JavaScript, but with optional type annotations for variables involved:

let n: number = 0; let x: number = 0; while (n < 3) { n++; x += n; }

Function Declarations

Functions in TypeScript are defined using the function keyword, with the addition of type annotations for parameters and return types:

function square(num: number): number { return num * num; }

This function takes a number as an argument and returns its square. The : number annotation after num ensures that the argument must be a number, and the annotation after the parentheses specifies the return type.
